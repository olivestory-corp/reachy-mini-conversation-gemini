# Reachy Mini 대화앱 (Gemini Live) — 실행 방법

Reachy Mini **Control 앱 없이**, 이 폴더만으로 대화앱을 실행합니다.
(공식 앱이 최신 버전에서 Gemini Live 지원을 없앴기 때문에, Gemini가 되던 **v0.8.0** 을 그대로 담았습니다.)

Gemini API 키는 이미 `.env` 파일에 넣어 두었습니다. **추가 설정 없이 클릭만 하면 됩니다.**

---

## ⛔️ 절대 주의 — Control 앱을 실행하지 마세요!

이 앱은 Reachy Mini **Control 앱 없이** 자체 데몬으로 돕니다. Control 앱을 켜면:
- 로봇 포트(`:8000`)가 **충돌해서 이 앱이 안 켜집니다.**
- Control 앱은 **자동 업데이트**로 Gemini 지원이 없는 최신 버전으로 바뀝니다(우리가 v0.8.0 을 따로 쓰는 이유).

👉 실행 전 Control 앱은 **완전히 종료(Quit)** 하세요. 이미 설치돼 있어도 **열지만 않으면** 됩니다.

## ⚠️ 실행 전 준비

1. 위 경고대로 **Control 앱 완전 종료.**
2. **로봇을 USB로 연결**하세요.
   (로봇 없이 화면만 테스트하려면 아래 "로봇 없이 테스트" 참고.)

---

> 💻 이 폴더는 Mac·Windows 둘 다 지원하지만, **Mac 사용자는 아래 🍎 Mac 부분만** 보시면 됩니다.

## 🍎 Mac  (← 사용하실 분은 여기)

### 준비 (최초 1회)
- **Python 3.12** 설치: https://www.python.org/downloads/
  ⚠️ 큰 버튼(최신 3.14 등) 말고, 페이지 아래 **"Looking for a specific release?" → Python 3.12.x** 를 받으세요.
  이 앱은 **3.12 검증**이며 3.13/3.14 는 의존 패키지 휠이 없어 설치가 실패합니다.
  (또는 터미널에서 `brew install python@3.12`)

### 실행
1. **`0.실행`** 폴더 안의 **`OSX_시작.command`** 더블클릭.
   - 첫 실행은 가상환경 생성 + 패키지 설치로 **몇 분** 걸립니다 (인터넷 필요). 다음부터는 빠릅니다.
   - 처음 열 때 "확인되지 않은 개발자" 경고가 나오면:
     파일 **우클릭 → 열기 → 열기**, 또는 시스템 설정 → 개인정보 보호 및 보안 → "확인 없이 열기".
2. 준비되면 로봇에게 **말을 걸면 됩니다.** (이 터미널 창에 로그가 보입니다.)

### 종료
- **`OSX_종료.command`** 더블클릭. (또는 시작 창을 닫으면 앱은 종료 — 데몬은 종료 파일로 정리)

## 🖥️ Windows  (Windows PC일 때만)

### 준비 (최초 1회)
- **Python 3.12** 설치: https://www.python.org/downloads/
  ⚠️ 큰 버튼(최신 3.14 등) 말고, 페이지 아래 **"Looking for a specific release?" → Python 3.12.x** 를 받으세요.
  이 앱은 **3.12 검증**이며 3.13/3.14 는 의존 패키지 휠이 없어 설치가 실패합니다.
  설치 화면에서 **"Add python.exe to PATH"** 체크 필수.

### 실행
1. **`0.실행`** 폴더 안의 **`WIN_시작.bat`** 더블클릭.
   - 첫 실행은 설치로 **몇 분** 걸립니다. 다음부터는 빠릅니다.
   - "Reachy 데몬" 이라는 **새 검은 창**이 하나 뜹니다 → **그 창은 닫지 마세요** (로봇 연결 창입니다).
2. 준비되면 로봇에게 **말을 걸면 됩니다.**

### 종료
- **`WIN_종료.bat`** 더블클릭. (대화앱 + 데몬 모두 종료)

> 참고: `WIN_시작.bat` 이 SmartScreen 경고를 내면 **추가 정보 → 실행**.

## 🤖 로봇 없이 테스트 (시뮬레이션 — 실험적)

> **이 앱은 로봇 하드웨어가 있어야 정상 동작합니다.** 데몬은 로봇에 연결되어야 켜지므로,
> 로봇이 연결돼 있지 않으면 "데몬 시작 실패"가 정상입니다. 아래 SIM 모드는 로봇 없이
> 설치·연결만 점검하는 **실험적** 옵션이며(MuJoCo 시뮬), 버전에 따라 불안정할 수 있습니다.

- **Windows**: `WIN_시작.bat` 을 메모장으로 열어 위쪽 `set "SIM=0"` → `set "SIM=1"` 저장.
- **Mac**: `OSX_시작.command` 를 텍스트 편집기로 열어 `SIM=0` → `SIM=1` 저장.

SIM=1 로 두면 첫 실행 시 MuJoCo 를 자동 설치하고 시뮬 데몬을 띄웁니다.
**실제 로봇을 쓸 땐 반드시 다시 `SIM=0` 으로** 되돌리세요.

## 🔑 Gemini 키를 바꾸려면

`.env` 파일을 열어 `GEMINI_API_KEY=...` 값을 새 키로 교체하세요.
(키는 https://aistudio.google.com/apikey 에서 발급)
⚠️ `.env` 는 키가 평문으로 들어있으니 **다른 사람과 공유하거나 인터넷에 올리지 마세요.**

## ❓ 잘 안 될 때

| 증상 | 해결 |
|---|---|
| 데몬 시작 실패 | Control 앱이 켜져 있는지 확인 → 종료. 로봇 USB 재연결. |
| `Python 3.12 필요` | python.org 에서 3.12 설치 (Windows는 PATH 체크). |
| 첫 실행이 너무 오래 | 정상입니다(패키지 다운로드). 인터넷 확인. |
| 응답이 안 나옴 | `.env` 의 `GEMINI_API_KEY` 가 유효한지 확인. |
| (Windows) `gi`/`gst` 오류 | 시작 창에서 `.venv\Scripts\activate.bat` 후 `pip install --reinstall reachy-mini` |
| 설치가 꼬임 | 이 폴더의 `.venv` 폴더를 삭제하고 시작 파일 다시 실행 (재설치). |
| 폴더 경로 문제 | 폴더 경로에 한글·공백이 없는 위치(예: `C:\reachy_win`)에 두기. |

```
│   ├─ OSX_시작.command   ← [Mac] 시작 (더블클릭)   ★ 사용하실 분은 이것

├─ .env                ← Gemini 키·설정 (공유 금지)
├─ 실행방법.md          ← 이 문서
├─ src/ profiles/ pyproject.toml … ← 대화앱 v0.8.0 소스

```

## 🧩 내부 동작 (참고)

- 시작 스크립트가 하는 일: ① 최초 1회 `.venv` 생성 + 패키지 설치(`uv sync --frozen`)
  ② CLI 데몬 `reachy-mini-daemon` 기동(:8000) ③ 대화앱 `reachy-mini-conversation-app` 실행.
- **데몬은 어디 깔리나?** `reachy-mini` 패키지에 데몬이 포함돼 있어, 설치 시 **이 폴더의 `.venv` 안에**
  같이 깔립니다. 시스템 전역도, Control 앱도 아닌 **폴더 자체에 자족적**입니다.
- **데몬 버전은?** SDK와 데몬이 같은 패키지라 항상 서로 맞습니다. `uv sync --frozen` 이 v0.8.0 이
  출시·검증된 버전(**reachy-mini 1.8.3**)을 락파일 그대로 설치하므로 버전 걱정 없습니다.
- Control 앱은 이 데몬을 GUI로 대신 띄워줄 뿐이라, **CLI 데몬으로 직접 띄우면 Control 앱이 필요 없습니다.**
- 백엔드 = Gemini Live (`.env` 의 `BACKEND_PROVIDER=gemini`), 모델 기본 `gemini-3.1-flash-live-preview`.
